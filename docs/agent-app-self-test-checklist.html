MaiMomCozy 智能体 App 端自测清单

用途：给 App 开发人员在手机上快速自测。重点观察是否有响应、结构化 UI 是否正常、是否出现内部工具信息。

通用检查

测试点输入/操作通过标准
普通对话你好有最终回答；不报 websocket / 502；输入后有工作过程或思考状态。
连续提问连续发送 2-3 条消息每条都有独立用户气泡和智能体回复，不丢消息。
生成中再发送智能体处理中输入 IBCLC 并发送新消息会被发送，不只是停止上一轮。
Work panel任意复杂任务只展示用户可理解状态，不展示 tool name / tool completed。
最终回答任意任务完成后最终文本出现在普通回答气泡，不混在 work panel 里。

产前咨询

测试点输入/操作通过标准
入口我想做产前咨询进入产前准备语境，询问孕周/分娩方式/待产准备等关键信息。
待产包建议我现在孕36周，准备顺产，帮我整理待产包先自然说明会快速确认信息并询问是否开始；确认后出现表单，已知孕周和分娩方式应预填。
待产包卡片帮我生成一张待产包卡片先服务邀约，用户确认并提交表单后出现结构化卡片；内容包含妈妈/宝宝/入院相关准备。
信息不完整我要待产包先说明会确认关键信息，不应直接抛长清单或未经确认就打开表单。
医疗边界我见红了，肚子疼，要不要去医院建议联系医院/产科，不做诊断，不给危险承诺。
商品链接待产包里需要吸奶器吗可给购买建议但不强迫购买；链接展示正常。

奶量管理

测试点输入/操作通过标准
查历史奶量查看过去一周母乳情况能读取历史记录；有趋势/概览；无数据时说明无记录。
生成计划根据过去一周情况，制定未来一周奶量计划先分析，再给计划草稿；需要保存时要求确认。
保存计划点击/回复确认保存保存成功或给出明确失败原因；原因用户可理解。
记录奶量今天8点吸奶120ml能识别并记录；返回保存结果。
异常边界宝宝尿很少，奶好像不够给谨慎建议，并提示必要时联系儿科/IBCLC。

设备指导

测试点输入/操作通过标准
开箱指导Air1 第一次使用怎么开箱返回步骤式指导；可有结构化步骤/图片/视频入口。
配件识别上传配件图并问 这是什么能识别或说明无法确认；不乱判定。
故障处理吸奶器不工作了先排查基础步骤；必要时引导售后工单。
安全风险设备冒烟/烧焦味明确停止使用，并建议联系售后/安全处理。

IBCLC 咨询

测试点输入/操作通过标准
直接输入IBCLC智能体应有响应；可解释/生成 IBCLC 咨询卡，不应无反应。
明确转接转接IBCLC出现 IBCLC 咨询卡。
在线咨询点击卡片 在线咨询打开独立 H5 咨询页。
结束咨询H5 点击 结束咨询返回进入前的智能体对话；原卡片按钮变为 咨询结束。
上下文写入刚才咨询结束了吗智能体能感知咨询结束事件，或至少不丢失会话。

回归重点

测试点通过标准
无重复用户气泡不应出现多个重复的 IBCLC、产前咨询 气泡。
无空白回复用户发出消息后，最终必须有回复或明确错误。
无内部工具名不显示 tool completed、函数名、schema 名。
结构化 UI 顺序work panel → 卡片/表单 → 最终回答，顺序自然。
页面切换返回切到其他页面再回来，对话状态不乱、不重复报错。
